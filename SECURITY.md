# Security Policy

## Supported Versions

Projekt jest aktywnie rozwijanym prototypem. Wsparcie bezpieczeństwa dotyczy bieżącej gałęzi `main`.

## Reporting a Vulnerability

Nie publikuj podatności publicznie w issue.

Zgłoszenie wyślij prywatnie z opisem:

- wpływu i scenariusza ataku,
- kroków reprodukcji,
- proponowanej poprawki (jeśli masz).

Do czasu utworzenia dedykowanego kanału kontaktowego użyj maintainera repozytorium przez GitHub profile/contact.

## Disclosure Process

- potwierdzenie przyjęcia zgłoszenia: do 72h,
- wstępna ocena wpływu: do 7 dni,
- plan naprawy i publikacja poprawki: zależnie od ryzyka.
